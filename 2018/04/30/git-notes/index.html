Git-notes

Overview

Here are some git commands in case of oblivion.
[TOC]

Basic operations

  • git init: create repository.
  • git add: add files from working directory to stage.
  • git commit -m "message": commit the changes from stage to master directory.

Version control

  • git status: check if there is any changes in current directory and have not been synchronized.
  • git diff filename: check the the changes of a file.

  • git log: see the history commits from log and get the commit id.

    • --pertty=oneline: beautify them in oneline.
    • HEAD: current version.(after last commit)
    • HEAD^: last version.
    • HEAD~n:last n version.

  • git reflog: get every history commits.

  • git reset --hard commit_id/pointer: back to specific version.--hard

Some concepts


Working directory
stage
master

Back from Changes

  • git checkout -- readme.txt:
    • If have not add, back to last commit.
    • If have been added, but have not been commit. git reset HEAD file could help you to unstage, which put your file back to working directory.
    • If have been commited, git reset --hard commit_id or git reflog to change another version, then push to origin.

Delete file

  • rm : delete file in the working directory.
  • git rm: delete the file in the repository.
  • git commit, git checkout.
  • git rm: delete files in .git. After you delete it. You can only revert it to the last version.(cannot roll back to other version)

Remote repository

  • git remote add origin remote-repository-address(if the remote repository is not empty, we should pull them in advance.)
  • git branch --set-upstream branch-name origin/branch-name: relate local branch and origin branch.
  • git push -u origin branch-name: -u make connection with the local branch and origin/branch branch.
  • git remote -v: get remote branch information.
  • Best practice: keep the same name in local repository and remote repository. git checkout -b branch-name origin/branch-name

Branch management

  • git checkout -b branch_name: create branch_name branch and switch to it.
  • git branch branch_name: create branch - branch_name
  • git branch: see current branch.
  • git merge branch_name: combine current branch with branch_name.
  • git branch -d branch_name: delete branch_name.
  • When different branch changed the same file and have conflicts, we need to edit the file directly and commit the change again. Then, we can safely delete the unnecessary branch.
  • git log --graph --pretty=oneline --abrev-commit.

Tag config

  • git tag v_num -m "info": create a snapshoot for current version.
  • git tag v_num prev_version
  • git show v_num
  • git tag -d v_num
  • git push origin <tagname>
  • git push origin --tags: push all tags which have not been pushed.
  • git push origin :refs/tags/v_num: delete remote tags.
  • git merge --no-ff -m "any message" branch: merge as a commit.

Bug fixing

  • When we have to interrupt our current work to start other work which have higher priority, we use git stash to save our working directory. Then, we can change our branch to fix bug. After complete the bug, we back to the prior branch, we can use git stash to revert.
  • git stash list
  • git stash apply
  • git stash drop
  • git stash pop

Ignore file

  • Ignore typies:
    • system files or created by dynamic system.
    • internal file when compiling.
    • sensitive configurations.

  • example

  • git add -f A.class: add .class by force.

  • git check-ignore -v file_name: check the conflicts in the .gitnore.
