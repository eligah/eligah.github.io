How to use Java log system

Overview

LOG系统是一套用于跟踪程序状态的机制. 区别与直接控制台输出, 在高并发的情况下效率高. 而且框架实现有利于方便查看, 保存信息.

The best practices of using Log

  1. 级别分类: DEBUG，INFO和ERROR. 将日志的重要程度进行分层, 不会造成混乱.
  2. 日志记录的内容(易用性): 日志级别, 时, 地, 人(thread), 物.
    • 日志级别：根据“一、只记录DEBUG，INFO和ERROR级别日志”，日志级别可取值有“DEBUG，INFO和ERROR”
    • 时间：日志记录行为发生时间
    • 地点：日志记录行为发生时所处类，根据具体情况可取“全限定类类名”或者“简单类类名”
    • 人物：进行日志记录行为的当前线程的线程名称
    • 事件：使用日志进行记录的事件内容
  3. 日志具有对机器可读性. linux系统下时常需要处理日志文件. 因此在基本的文件命令(grep, sed, cut, tail)使用时, 日志需要具有良好的格式.

log4j,logback,slf4j的关系

  • Log4j是Apache的一个开源项目，它为日志的输出提供了完美的解决方案，利用它可以在项目中高效的控制日志和归档.
  • slf4j是日志框架的抽象, 支持使用遵循一套日志接口的日志框架快速切换.
  • logback 是log4j 对于slf4j的适配层, 性能上优于log4j.

logback 配置文件的使用

resource 目录下的logback.xml, 如果需要覆盖默认配置, 建议命名logback-spring.xml.

Configuration

<configuration scan="true" scanPeriod="60 seconds" debug="false">  

</configuration>
  • scan属性如果设置为true，在配置文件发生变化时，会自动加载，默认时false。
  • scanPeriod属性只有scan属性设置为true才会生效，表示扫描周期，默认为60s，注意要设置时间单位。
  • debug为true时可以打印出logback内部日志信息，从而实时查看logback运行状态。

Property define

方便引用路径${LOG_PATH}, 设置时间戳, 定义上下文, 区分不同的程序.

<property name="APP_Name" value="LightQA" />
<property name="LOG_PATH" value="logs"/>
<property name="LOG_ARCHIVE" value="${LOG_PATH}/archive"/>
<timestamp key="timestamp-by-second" datePattern="yyyy-MM-dd HH:mm:ss"/>
<contextName>${APP_Name}</contextName>

Appender

<appender name="File-Appender" class="ch.qos.logback.core.FileAppender">
    <!-- append 是否追加日志 默认true -->
   <file>${LOG_PATH}/logfile-${timestamp-by-second}.log</file>
   <encoder>
       <!-- 日志的输出格式 -5level 日志级别, logger{}输出logger名称-->
       <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} -%msg%n</pattern>
       <!-- 输出pattern -->
       <outputPatternAsHeader>true</outputPatternAsHeader>
   </encoder>
</appender>

rollingPolicy 设置每天生成日志

<rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
       <fileNamePattern>${LOG_ARCHIVE}/rollingfile.%d{yyyy-MM-dd}.log</fileNamePattern>
       <!--归档文件保留周期，这里30表示30天，超过后最老的日志被删除-->
       <maxHistory>30</maxHistory>
       <!--定义日志最大量，超过了最老的日志将被删除，这里是10G-->
       <totalSizeCap>10240MB</totalSizeCap>
   </rollingPolicy>

Logger

功能: 特定的类设置单独的日志输出
添加appender-ref ref = “appender-name” 为logger添加属性

<logger name="com.netease.kaola.gaoshengli.learnspringboot.controller" level="info">
   <!--日志记录文件的appender-->
   <!--<appender-ref ref="File-Appender"/>-->
   <!--归档文件的appender-->
   <!--<appender-ref ref="RollingFile-Appender"/>-->
   <!--选择异步方式-->
   <appender-ref ref="Async-Appender"/>
</logger>

root 相当于特殊的logger, 普通的logger从root节点继承属性.
